ArcGis Network Analysis - Service Area from Open Street Map (OSM) Data

Downloading the OSM Data

To being you’ll need some street level data to build a network dataset. Open Street Maps is open source and in most countries, provides quality road data. It’s worth noting that your eventual service area quality will only ever be as good as the initial map data.

Furthermore, It is possible to download and OSM data file from the Open Street Map website for free.

https://www.openstreetmap.org/export

 

You can also manually select areas on the left-hand side however this export will only work at low zoom levels (and you may otherwise get an error message if your area is too large). This was done by OSM on purpose to reduce the strain on their servers and there are ways to export large amounts of data by using a third-party server.

You’ll still need to select your area

 

and press the export button.

If a download starts with a .osm file then it means you’re in the clear and can skip to the next section.

If you get an error then you’ll most likely need to use a third party download server. You’ll still have a URL with the area you selected and the coordinates come in handy when downloading externally.

My URL for the Canberra area looks something like this:

https://www.openstreetmap.org/api/0.6/map?bbox=149.0778%2C-35.3346%2C149.1808%2C-35.2853

 

External Server URL

The most common server today is hosted in Germany however there are other options listed on the OSM Wiki

The URL for the server is:

http://www.overpass-api.de/api/xapi_meta?*[bbox=?,?,?,?]

and the “bbox” in the URL above (which stands for bounding box) requires the same coordinates that the export link gave you above

You now need to enter your coordinates into that link and download the file. For my Canberra example it would look something like

http://www.overpass-api.de/api/xapi_meta?*[bbox=149.0778,-35.3346,149.1808,-35.2853]

This will then download a file called “xapi_meta” to your downloads and you’ll need to add a .osm extension before continuing in ArcMap.

To add a file extension you’ll need to open the files location (most likely in your download folder), click on it, click rename and add  .osm to the end of the file name.

Adding ArcGis Editor For Open Street Map (Open Street Map Toolbox)

 

To load the OSM toolbox into ArcMap you’ll need to first install the Open Street Map toolbox. It can be downloaded for free from the ESRI website and the download link is located at the bottom the page here. It’s worth pointing out you’ll need to ensure you download and install the toolbox for your version of ArcMap (10.6, 10.5 etc).

After installing the OSM toolbox it should appear in ArcMap like the following

Note : If this doesn’t occur it’s also possible to manually add the OSM toolbox by right clicking and pressing “Add Toolbox…”

You’ll then be asked to locate the toolbox .tbx file.

If should be located in Toolboxes à System Toolboxes à OpenStreetMap Toolbox.tbx

 

Creating a File Geodatabase

Before you begin to create a map you’ll need to create a Geodatabase to store your road, point and shape file data that gets exported from the .osm file you downloaded earlier.

In your folder connected to ArcCatalog you can right click and select New à File Geodatbase


Ensure you name the Geodatabase something you can remember and once successfully created you should see a new file listed.

 

Loading the OSM File into the Geodatabase

 

You can use either ArcCatalog or ArcMap to load the OSM file. I am using ArcCatalog however the instructions aren’t any different between the two.

For the next steps you should turn off background processing. This will speed things up and ensure that all processes work correctly.

To turn off background processing select Geoprocessing à Geoprocessing Options…

 

Ensure that background processing is not enabled

 

Now in the OSM Toolbox select “Load OSM File”.

Here you will need to select your .osm file you saved earlier for the OSM File and select a location within your recently created File Geodatabase for the Target Feature Dataset.

I’ve called my Target Feature dataset “CanberraData” however it doesn’t matter what it’s called. Once both of these fields have been filled in you can then press OK. It will take some time to load and will end with a window that says “Completed”. You can close this window and you’ll now see that the Geodatabase now has line, point and shape files inside.

 

 

Creating a network Dataset

Locate the OSM Toolbox. If it doesn’t appear you might need to add it as seen in the instructions earlier. The tool you are now looking for is the “Create OSM Network Dataset”

You only need to select files for the first two inputs. The third will automatically be generated.

The First Input needs the Dataset you created in your Geodatabase in the step above. I called mine “CanberraData” however yours might have a different name. The second input requires a Network Configuration File. This file should have automatically been loaded when you installed the OSM Toolbox and is located in “C:\Program Files (x86)\ArcGIS\Desktop10.6\ArcToolbox\Toolboxes\ND_ConfigFiles\DriveGeneric.xml”


Press OK and wait for the process to create a network dataset. Once the process is completed you should get a message like the following:

You can close that window.

You need to add all the files with _nd that are located in your geodatabase into the ArcMap file.

 

You may get a message about adding all feature classes – select yes to this pop up

Adding Locations

To generate a service area we first need to tell ArcMap what locations we want to base the service area on. You can load external Shapefile, Layer or even KML files from Google Earth into ArcMap and use them for the Network Analyst however for this guide I’ve created a new layer and added my own points to the map.

To add a new shapefile open ArcCatalog. Locate your working folder and right click à New à Shapefile…

Note: It’s recommended you don’t create the shapefile inside the Geodatabase you are using.

I’ve called my Shapefile points however the name doesn’t matter. You will also need to set a spatial reference for the new Shapefile. As the OSM data uses WGS_1984 it’s best if you also use this for the point data as well.

You can now return to ArcMap and add the Shapefile to your map. Currently it’s empty and you’ll need to start editing to add points to the file.

I have my Editor Toolbar docked along the top of ArcMap however if it doesn’t appear you can find it in Customize à Toolbars à Editor

 

ArcMap will ask you which layer you’d like to edit. Here you’ll need to select the shape file you recently created and added.

You’ll also need to ensure you’ve selected the create features option from the Editor Toolbar

And after ensuring “points” is selected you can then click on the map to add points.

You can add a few points to your map and once complete you need to save your edits (Save Edits) in the toolbar and then select stop editing.

I added 4 points which are now visible on the map and in the Attribute Table (you can also change the point symbology to make it easier to view on a map).

 

Creating the Service Area

Now you have you map set up it’s now possible to create a service area.

Note: If you have not yet saved your map it is worth doing so before creating the service area.

Firstly, you’ll need to ensure that the Network Analyst tool is switched on in the Extensions by Selecting Customize à Extensions

You’ll also need to ensure you have the network analyst toolbar enabled in the Customize à Toolbars menu.

ArcMap has automatically recognised my CanberraData network dataset and now it’s possible to select create new service area:

 

Upon selecting New service area you’ll notice that a “Service Area” layer has been added however before calculations are made you’ll need to configure your service area. This can be done through the network analyst window

 

The Network Analyst window manages the distance, polygon and point settings of your service area.

You’ll first need to add your points to the facilities by right clicking on “Facilities” and select “Load Locations”

The load locations window will look something like this:

It important to ensure that you are loading the shapefile containing the locations you plan to analyse.

You’ll notice that the Facilities now displays the number of points that have been loaded into the network analysis.

The next step requires the “Service Area Properties” window

The two most important tabs are the “Analysis Settings” and the “Polygon Generation” as these determine the eventual output.

Impedance : There are two options for a service area. Either calculating by time or distance. As I’m trying to find shortest distance I’ll use the distance option. It’s worth noting that OSM data doesn’t always have speed data which can make the time calculations inaccurate in some areas.

Default Breaks : Here you enter the size of the service area. In this example I’ll create two polygons around each point, one at 500 meters and one at 1000 meters. It’s possible to just enter one distance/time or several by using commas to separate.

Direction : This can be used if calculating service areas in locations with several one-way streets where the distance may make a difference depending on the direction used.

U-Turns at Junctions : In Canberra U-Turns are allowed only at signed intersections and you can select an option that suits your local area.

 

In the Polygon Generation tab you can select your desired output polygons.

Polygon Type : Essentially the caculation of the polygons. The detailed option is better but also the more time required

Trim Polygons : This is the distance that the polygons are from the road. A higher distance will mean that polygons cover large blocks or areas without streets but this can also be less accurate.

Once you’ve selected your desired options you can click OK.

Finally to generate your Polygons or Service area you need to select “Solve” from the Network Analyst Toolbar

It may take a while to generate depending on area but eventually the new service area will appear on your map.

 

You can alter the Polygons symbology to create a map with more meaning to each location.
